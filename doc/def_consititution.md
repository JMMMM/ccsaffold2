我打算做一个AI脚手架，在我每次进行新的AI编程时，都可以选择这里面的功能运用到新项目中；

# 技术选型
本项目优先用nodeJS，如果无法实现再考虑python，python用的是python3.9；

# 参考文档
在进行claude code 功能组件开发时，必须参考claude code 官方文档，你可以调用context7 mcp先了解文档，例如我想开发hooks，那么你必须通过contex7了解claude code hooks的相关内容;
了解后在进行开发

# 编码规范：
1.可复用原则和单一功能原则，高内聚低耦合；
2.代码必须是人类可读懂的，保持简单直接高效，在可读性和性能上，可以优先选择可读性；
3.采用TDD开发模式；
4.因为需要兼容多个平台，如果nodejs能兼容3个平台就不要创建ps1和sh脚本了，直接在nodejs中处理兼容性问题;

# 信息安全：
1.必须保障项目不不能带有用户个人信息，系统信息等敏感信息；
2.必须保障项目不包含任何违法违规内容；

# 功能要求
- 项目中功能必须可以单独被使用例如我迭代了10个功能，那么我可以自主挑选3~4个功能进行使用；
- 所有功能都在feature中开发，feature，每个功能有.claude下的文件夹，例如SKILLS,COMMANDS,AGENTS,HOOKS等文件夹，维护在feature下的对应功能名下；
- 某些功能实现后，可能会有文件记录和维护等要求，这些要记录的文件默认放在.claude目录下,例如会话记录文件放在.claude/conversations/目录下，技能库文件放在.claude/skills/目录下等；
- 在功能实现时需要考虑维护工作目录这类变量，避免通过..的方式访问上级目录，确保功能的独立性和安全性；
- 功能实现时需要考虑到用户的使用习惯，尽量提供简单易用的接口和操作方式；
- 功能实现时需要考虑到性能，确保功能的高效运行；

# 兼容性
1.功能必须兼容不同操作系统和环境，例如Windows、Linux、macOS等；

# 其他
涉及到宪章文件的修改，需要在根目录下同步更新，保持两个文件一致性；
